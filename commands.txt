To make psycopg2 use OpenSSL 1.0.0:

export DYLD_LIBRARY_PATH=/usr/local/Cellar/openssl/1.0.1c/lib

or:

sudo ln -s /Library/PostgreSQL/9.2/lib/libssl.1.0.0.dylib /usr/lib
$ sudo ln -s /Library/PostgreSQL/9.2/lib/libcrypto.1.0.0.dylib /usr/lib

To dump database for Heroku:

/Library/PostgreSQL/9.2/bin/pg_dump --host localhost --port 5432 --username "postgres" -Fc --no-acl --no-owner --verbose --file "/Users/jhorneman/Desktop/data.dump" "gdcal-dev"

To import data dump into Heroku database:

heroku pgbackups:restore DATABASE 'http://www.intelligent-artifice.com/data.dump' --confirm gameconfs

To grab database backup from live server on Heroku:

heroku pgbackups:capture --expire && curl -o latest.dump `heroku pgbackups:url`

(Source: https://devcenter.heroku.com/articles/heroku-postgres-import-export)

To import Heroku database backup into local Postgres:

pg_restore --verbose --clean --no-acl --no-owner --host localhost --port 5432 --username "gdcal-dev" --dbname "gdcal-dev" latest.dump

(The user is important! See: http://www.softr.li/blog/2012/07/25/postgresql-schema-owner-altered-during-dump-prevent-access-from-rails)

To create database:

CREATE USER "gdcal-dev" WITH PASSWORD "<password>";
CREATE DATABASE "gdcal-dev";
GRANT ALL PRIVILEGES ON DATABASE "gdcal-dev" to "gdcal-dev";

To list backups on Heroku:

heroku pgbackups

(See also: https://devcenter.heroku.com/articles/pgbackups)

To upload to Heroku:

git push heroku master

To install PostgreSQL locally:

- OS X has a crippled version of Postgres pre-installed.
- Currently, we have the EnterpriseDB distribution installed.
- In /Library/PostgreSQL/9.2/ is pg_env.sh, a shell script that points Postgres commands to the installed version, not the crippled version.
- In /Library/PostgreSQL/9.2/ is uninstall-postgresql, an uninstaller.
- There is a PostreSQL 9.2 folder in /Applications.
- Data is stored in ~/dev/PostgreSQL/9.2/data.

"dyld: Library not loaded: /usr/lib/libpq.5.dylib" - do this to fix it:

sudo ln -s /usr/lib/libpq.5.3.dylib /usr/lib/libpq.5.dylib

(See this: http://superuser.com/questions/352005/os-x-lion-web-sharing-cant-start)
